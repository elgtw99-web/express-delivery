# 採購系統・下一階段詳細規劃

> 主題：把「到貨驗收 → 成本計算 → 進貨財務對帳」串成一站式，讓員工一次輸入、全流程沿用，系統自動算完，人只做確認與例外處理。

---

## 0. 目前系統覆蓋（起點）

| 採購流程階段 | 系統現況 |
|---|---|
| 1 詢價 / 報價 | ⬜ 尚未做 |
| 2 客戶下單（採購三聯單） | 🔶 有訂單建檔，無三聯單 |
| 3 尋品（新品） | ⬜ 尚未做 |
| 4 下採 · 國際跟單 | ✅ 物流追蹤 ＋ 免費即時 LINE 推播（訂閱推送，不花額度） |
| 5 到貨驗收（進貨單） | ✅ 實收／短少／多出／瑕疵／損毀，自動出進貨單、推 LINE |
| 6 成本計算 | 🔶 成本清單雛形（RMB＋台幣、運費單列），缺自動平攤與匯率 |
| 7 進貨財務對帳 | ⬜ 尚未做 |

**系統已保存的資料（不用重做，直接接下去用）**

- 訂單：供應商、訂單編號、快遞公司、快遞單號、手機末四碼、地區、幣別
- 產品（一筆多項）：品名、採購數量、單價、實收數量、瑕疵數、損毀數
- 金額：貨品小計、國內運費、國際運費（海運／空運）、匯率、實付金額
- 到貨：到貨日期、收貨人、備註、是否到齊
- 物流：目前狀態、最新軌跡、時間軸

---

## 1. 設計原則（減少人力的核心）

1. **一次輸入、全流程沿用**：採購時輸入的品項／數量／單價／運費，到貨、成本、對帳都不再重打。
2. **自動計算優先**：系統把能算的都算好，人只做「確認」與「處理例外」。
3. **角色分工清楚**：採購、倉庫、會計、業務各自只碰自己那段。
4. **金額雙幣別、單一真實來源**：所有數字 RMB＋台幣並列，全系統以同一筆訂單資料為準。

---

## 2. Phase A — 成本計算（自動平攤 ＋ 月結）〔建議先做〕

### A1 目標
資料進來後，**每件商品的實際成本（台幣）自動算好**，會計月底一鍵匯出，不用再手拉 Excel。

### A2 成本組成（每一筆訂單）
- 貨品成本（RMB）＝ Σ（單價 × 數量）；淘寶訂單以「實付金額」為準
- 國內運費（RMB）
- 國際運費（海運／空運）
- 其他共同費用（報關費、雜費）— 選填

### A3 匯率處理
- 每筆訂單可自填匯率；**未填者自動套用「當月匯率」**。
- 新增「月匯率表」：一個月一格（例如 2026-07 = 4.45），會計月初填一次即可。
- 成本清單同時顯示 RMB 原價與換算後台幣。

### A4 平攤邏輯（把共同費用分到每一件）
共同費用 = 國內運費 ＋ 國際運費 ＋ 其他費用。兩種平攤方式，每筆訂單可選（預設「依金額」）：

- **依件數**：共同費用 ÷ 訂單總件數 × 該品件數
- **依金額**：共同費用 × （該品貨值 ÷ 訂單總貨值）

**單件實際成本（台幣）** ＝（該品貨值 ＋ 該品分攤到的費用）× 匯率 ÷ 該品數量

### A5 需新增的資料欄位
- 訂單層：`otherFee`（其他費用）、`apportionMethod`（'qty' | 'amount'）
- 產品層：`allocatedCost`（分攤後總成本 TWD）、`unitCost`（單件成本 TWD）
- 全域：`monthRates`（月份 → 匯率對照）

### A6 介面
- 訂單「成本」面板：自動顯示每件分攤後成本（台幣），可切換平攤方式即時重算。
- 月結頁（升級現有成本清單）：選月份 → 自動列出每筆／每件成本、含分攤與單件成本 → 匯出 Excel／列印。

### A7 誰做什麼（workflow）
- 採購：匯入訂單時已帶單價、國內運費、實付。
- 採購／倉庫：到貨後補國際運費（海運／空運）。
- 會計：月初填當月匯率 → 月底按「月結」一鍵出表。

### A8 驗收標準
- 隨機抽 3 筆訂單，人工試算單件成本與系統一致（誤差 < 0.01）。
- 切換「依件數／依金額」數字正確重算。
- 未填匯率的訂單能正確套用當月匯率。

## 3. Phase B — 進貨財務對帳（新模組）

### B1 目標
**系統實付 vs 供應商／淘寶帳單自動比對**，差異一眼看出，付款狀態可追蹤，月底出對帳表給財務簽核。

### B2 對帳來源
- 系統實付：每筆訂單的實付金額／貨品成本。
- 外部帳單：供應商對帳單、淘寶帳單（Excel 匯入）。

### B3 比對邏輯
- 以 訂單編號／快遞單號／品項 對應系統訂單。
- 差異標記：✅一致、🔺多付、🔻少付、⚠️系統有・帳單無、⚠️帳單有・系統無。
- 容差可設（例如 ±1 元內視為一致）。

### B4 付款狀態
- 每筆：未付 / 部分付 / 已付；記錄付款日、付款方式、已付金額。
- 未付、到期提醒自動推 LINE。

### B5 對帳表輸出
- 月對帳表：依供應商彙整、差異清單、應付總額。
- 匯出 Excel 給會計／財務簽核。

### B6 需新增的資料欄位
- 訂單層：`billAmount`（帳單金額）、`payStatus`、`paidAmount`、`paidDate`、`payMethod`、`reconcileNote`

### B7 介面
- 對帳頁：匯入帳單 → 自動比對 → 差異以紅字標示 → 逐筆標記付款狀態。

### B8 驗收標準
- 匯入一份帳單，系統正確標出所有差異筆數。
- 付款狀態變更後，月對帳表的「應付總額」同步正確。

---

## 4. Phase C — 到貨驗收閉環（業務 LINE 確認）

讓到貨後的「業務確認」也在 LINE 完成，形成閉環，不用另外回報。

- 到貨推 LINE 後，業務在群組回「確認 訂單號」→ webhook 標記 `confirmed / confirmedBy / confirmedAt` → 該筆結案。
- 群組打「待確認」→ 列出所有已到貨但業務尚未確認的訂單。
- 需新增欄位：`confirmed`、`confirmedBy`、`confirmedAt`。

---

## 5. 前段流程（選配，之後再串）

詢報價記錄、尋品比價、採購三聯單建檔——把整條採購鏈補齊，屬較後期。

---

## 6. 建議實作順序與里程碑

- **M1 成本自動平攤**（A2–A5）：單件成本自動算出。
- **M2 月結升級**（A3、A6–A7）：月匯率表 ＋ 一鍵月結成本清單。
- **M3 財務對帳**（B）：帳單匯入、自動比對、付款追蹤、對帳表。
- **M4 驗收閉環**（C）：業務 LINE 確認結案。

---

## 7. 角色與權限（多人協作）

| 角色 | 主要操作 |
|---|---|
| 採購 | 建立／匯入訂單、補國際運費 |
| 倉庫 | 登記到貨、驗貨查收（實收／瑕疵／損毀）、出進貨單 |
| 會計 | 填月匯率、月結成本清單、財務對帳、標付款 |
| 業務 | LINE 查詢物流、確認到貨結案 |

---

## 8. 一站式減工重點回顧

採購輸入一次 → 物流自動追蹤並即時推 LINE → 倉庫驗貨一鍵出進貨單並通知 → 系統自動算出每件成本（台幣）→ 會計月結一鍵出表 → 帳單匯入自動對帳、追付款。全程只在「確認」與「例外」處需要人。
